# Lead Management System

## Overview
Monorepo with:
- **Backend**: Node.js/Express + MongoDB API (`backend/`) on port ${PORT || 4000}
- **Frontend**: Next.js 16 App Router dashboard (`frontend/`)

## Local Development
```bash
# Terminal 1 - Backend
cd backend
npm install
npm run dev  # or npm start

# Terminal 2 - Frontend  
cd frontend
npm install
npm run dev  # http://localhost:3000
```

## Render Deployment

### 1. Backend (Web Service)
- **Root Dir**: `/` (clones repo root)
- **Runtime**: Node.js
- **Build Command**: `cd backend && npm install`
- **Start Command**: `cd backend && npm start`
- **Environment Variables**:
  ```
  MONGODB_URI=mongodb+srv://...  # Render External DB or Atlas
  JWT_SECRET=your-super-secret-key
  N8N_WEBHOOK_URL=...  # optional
  ```
- **Instance Type**: Free/Starter

### 2. Frontend (Static Site)  
- **Root Dir**: `/frontend`
- **Build Command**: `npm install && npm run build`
- **Publish Directory**: `.next` (auto-detected for Next.js)
- **Environment Variables**:
  ```
  NEXT_PUBLIC_API_URL=https://your-backend.onrender.com  # Backend URL
  ```
- **Custom Domain** optional

### Render URLs
- Backend API: `https://your-backend-name.onrender.com/leads`
- Frontend: `https://your-frontend-name.onrender.com/agent-dashboard`

## Production Notes
- MongoDB: Use Render's PostgreSQL + mongoose adapter or Atlas.
- Free tier: Backend sleeps after inactivity (~15min cold starts).
- Scale: Upgrade to Starter for always-on.

## API Endpoints
```
GET  /leads          # All leads
GET  /leads/:id      # Single lead  
GET  /my-leads/:chatId  # Agent's leads
GET  /agents         # All agents
POST /agents         # Create agent (auto password)
PUT /assign          # Assign lead
PUT /update-lead     # Update status/notes
PUT /unassign-lead   # Unassign to queue
/auth/login, /auth/signup
```
